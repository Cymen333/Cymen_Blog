## C 与 C++ 是什么
C由丹尼斯·里奇 (Dennis Ritchie) 于 1972 年在贝尔实验室创建。

C++由 Bjarne Stroustrup 开发，在 2011 年、2014 年、2017 年和 2020 年进行了 4 次重大更新，分别为 C++11、C++14、C++17、C++20。

C是一种通用编程语言，面向过程，是计算机科学领域的基础语言，也算是所有语言的鼻祖，这也导致了C始终有着一群忠诚的信仰者。~~(!!!C永生不死!!!)~~

C++是一种跨平台语言，面向对象，可用于创建高性能应用程序，同时也具有可移植性，可用于开发可适应多个平台的应用程序。

C++是作为C的扩展所创作的，这也造就了它们的极其相似的语法与硬件级的代码效率。

## 为什么学习 C 与 C++
就我个人的学习体会而言，原因有两点：

**1. 榨干计算机价值**

现代语言为了降低门槛，把内存管理、垃圾回收这些脏活累活全包了。而 C/C++ 是少数几门能让我直接操作内存地址、亲自控制字节流向的语言。学它们，能够学到**计算机到底是如何工作的**。

(关于我：我很珍惜钱，在我小时候，计算机就是我能接触到最贵的东西了，但我的家人几乎都不用计算机，只有我一周能够碰2小时，把计算机放在家里就是落灰，看着计算机的开机一次比一次慢，在后面我甚至是抱着机箱祈祷它能够开机。那时发生的一切令我十分心疼，在我眼里就是钱财的流失和价值的浪费，这导致了我现在的执念。)

**2. 极致的性能控制**

可以满足我畸形的控制欲。

## 因 C 与 C++ 而产生的互联网现象
因为这两门语言极度陡峭的学习曲线和“硬核”的属性，它们在程序员圈子里衍生出了许多有趣的迷因 (Meme) 和互联网现象。

### 鄙视链与“原教旨主义”
在很长一段时间里，C 程序员和 C++ 程序员互相看不顺眼。
- C 语言的信徒认为：C++ 过于臃肿，加了太多没用的特性，不够纯粹。他们信奉“少即是多”。
- C++ 程序员则认为：C 还在用原始的工具造轮子，而 C++ 有 STL (标准模板库)，有面向对象，不仅快，还能构建庞大的现代系统。

*(甚至连 Linux 之父 Linus Torvalds 都曾公开发表过对 C++ 的激烈批评，这更是加剧了这种充满火药味的“文化冲突”。)*

C++ 的语法更新实在太快、太复杂了。从 C++11 到 C++20，它几乎变成了一门新语言。互联网上最著名的段子莫过于：
> “如果你觉得自己精通 C++，那说明你还没入门。”
